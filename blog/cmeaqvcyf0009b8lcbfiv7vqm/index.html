使用 argparse 建構 Python CLI 工具的核心運作邏輯

前言

隨著自動化與工具化需求增加，命令列工具（CLI）在程式開發中扮演重要角色。Python 的 argparse 模組提供靈活且強大的參數解析能力，讓開發者能夠快速建構可擴充且易維護的 CLI 應用。本文將以 HTML 轉 Markdown 工具為例，整理 argparse 的主要架構與運作邏輯，幫助讀者掌握命令解析的核心知識。

運作邏輯

初始化 Orchestrator

Orchestrator（主解析器）負責統籌所有命令列參數的解析，並建立 CLI 工具的主架構。

  • ArgumentParser 物件是所有參數解析行為的核心。
  • description 用於提供工具全域描述，顯示於 help 訊息。

初始化代理

代理（subparsers）負責各自命令的參數解析，能夠區分並處理不同功能模塊。

  • 各代理解析器（如 convert_parser）專責特定命令的參數。
  • 可針對功能需求自由擴充多個命令解析器。

初始化 SupervisorAgent

在此架構下，SupervisorAgent可視為管理指令執行的調度者，負責依命令類型呼叫對應函式。

  • 透過 set_defaults(func=...) 將命令與處理函式綁定，確保後續能自動調用。

處理用戶請求

用戶請求被主解析器解析後，會自動分派至對應命令的處理函式。

  • parse_args() 解析整體命令列參數（包含主命令與子命令）。
  • 依據 args.command 判斷是否有指定子命令，否則僅顯示說明。
  • 由 args.func(args) 依解析結果執行專屬處理邏輯。

主函數

主程式作為 CLI 工具入口，負責啟動解析流程並處理互動。

  • 將所有初始化、代理配置整合於主函數，確保工具能正確運作。

程式運行流程

  1. 建立主解析器（Orchestrator）
    初始化 ArgumentParser，設定全域說明。

  2. 新增代理（命令解析器）
    透過 add_subparsers 與 add_parser 定義各命令及其參數。

  3. 綁定處理函式（SupervisorAgent）
    使用 set_defaults，將命令綁定至對應的處理函式。

  4. 解析與分派用戶請求
    使用 parse_args() 解析參數，並由 args.func(args) 執行命令。

備註：

  • 對於多命令、多功能需求，可於「初始化代理」區段增添更多解析器。
  • 可根據業務需要，於處理函數實作錯誤管理、輸出格式化等進階功能。
  • 所有程式碼皆可直接複製並於 Python3 環境執行。
