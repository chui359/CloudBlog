EKS 子網路與 NAT Gateway 出口 IP 更換問題整理

前言

在 AWS EKS（Elastic Kubernetes Service）架構下，節點（Node）通常部署於私有子網路（Private Subnet），需藉助 NAT Gateway 存取外部網路。本篇文章將針對如何確認 EKS 節點所使用的 NAT Gateway 及更換出口 IP 進行重點整理與範例說明。

問題

如何確認 EKS 叢集中每個節點目前經由哪一個 NAT Gateway 存取外部網路，以及如何安全且有效地更換 NAT Gateway 的出口 IP？

範例

1. 查找 Node 所屬 Subnet 與 NAT Gateway

步驟一：查詢 Node 所在 Subnet

  • 於 AWS 控制台進入 EKS 叢集，點選「Compute」->「Nodes」，檢視 EC2 實例。
  • 查看「Network interfaces」中的 Subnet ID。

步驟二：查詢 Subnet 的 Route Table

  • 於 VPC 控制台，搜尋該 Subnet ID，確認其所關聯的 Route Table。
  • 在 Route Table 內查詢 0.0.0.0/0 路由規則，確認是否指向 NAT Gateway（nat-xxxxxx）。

步驟三：查詢 NAT Gateway 的 Elastic IP

  • 於 VPC 控制台點選「NAT Gateways」，查詢相應 NAT Gateway 的 Elastic IP。

範例程式碼：Node 端確認出口 IP

回傳的即為 NAT Gateway 的 Elastic IP。

2. 更換 NAT Gateway 的出口 IP 流程

  • 於 VPC 控制台建立新的 NAT Gateway並綁定新的 Elastic IP。
  • 修改私有子網路 Route Table，將 0.0.0.0/0 的目標改為新的 NAT Gateway。
  • 完成後可再次於 Node 端確認出口 IP 是否已更換。

EKS 子網路與 NAT Gateway 概念補充

  • 私有子網路（Private Subnet）：無法直接連接 Internet，需透過 NAT Gateway 存取外部網路。
  • 公開子網路（Public Subnet）：可直接連接 Internet，常用於部署 Load Balancer。
  • NAT Gateway：部署於公開子網路，負責將私有子網路流量導向 Internet 並隱藏內部資源，出口 IP 為 Elastic IP。

架構示意圖

備註

  • 若有多個 NAT Gateway，建議分散部署於多個可用區以提升高可用性。
  • 更換 NAT Gateway 後，請確認所有相關 Route Table 均已更新。
  • 可搭配 Security Group、Network ACLs 進行進階流量控管。
