前言

當用 VS Code 連接（attach）到 Docker 容器內的 Python 程式（使用 debugpy）時，遇到「breakpoint in file that does not exist」錯誤的調查與解決方法。

問題描述

情境描述：

用 VS Code 設定斷點，但 attach 到 Docker 容器時，斷點沒有生效，且出現「breakpoint in file that does not exist」的提示。

主要原因：

通常是 VS Code 的 pathMappings 沒有正確對應本地檔案和容器內的實際路徑，或 mapping 的順序設錯，導致覆蓋了正確的設定。

處理步驟

對應 mapping：

  • helper.py（容器內）：/app/libs/utils/helper.py
  • helper.py（本地）：C:\Users\你的位置\libs\utils[helper.py](http://helper.py/)

常見問題與建議

  • 如果還是出現“file does not exist”，請複製 debug console 顯示的 remote path，檢查 pathMappings 是否正確對應。
  • 如果容器內 PYTHONPATH 或 sys.path 有更動，請用 python -c "import sys; print(sys.path)" 檢查。
  • 專案結構建議統一，方便維護與設定 mapping。

結語

依照以上步驟檢查並調整設定後，VS Code 應能正確連結你本地的檔案與容器內的程式碼，確保斷點可正常命中。如果需要進一步協助，請提供 debugpy 顯示的 remote 路徑，我可以協助你調整 launch.json 設定。
