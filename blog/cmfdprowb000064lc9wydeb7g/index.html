前言

開發者常以 Docker Compose 管理多服務架構，本文將實作如何於 Docker Compose 環境下，結合 debugpy 與 VSCode 完成 Python 服務的遠端偵錯，提升開發效率。

處理問題

本篇文章主要目標：

  • 讓 Docker Compose 啟動的 Python 服務（如 FastAPI、CLI server）能被 VSCode 斷點偵錯。
  • 解決 Docker 環境下程式碼路徑、端口映射與 debugpy 配置的常見疑難。

預計處理辦法：

  • 在服務啟動時結合 debugpy，並於 VSCode 使用 attach 模式連線。
  • 配置正確的端口映射與程式碼 volume，確保斷點可用。
