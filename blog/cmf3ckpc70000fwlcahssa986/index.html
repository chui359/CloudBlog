FastAPI 取得 Cookie 實作與前後端互動重點知識整理

前言

在現代 Web 開發中，前端與後端之間的資料互動方式多元，而 Cookie 是常見的狀態維護與資料傳遞技術。本文針對在 FastAPI 後端如何取得前端以 Cookie 傳遞的參數，並釐清前端使用 Cookie 的理由及傳遞流程，協助開發者建立完整的應用知識。

問題及回答

問題1：如何在 FastAPI 中取得前端透過 Cookie 傳遞的參數（以 ETOK、EMAIL 為例）？

可以利用 FastAPI 的 Cookie 參數輕鬆取得前端傳來的 Cookie 值。例如：

  • etoken: Optional[str] = Cookie(None) 會自動取得 Cookie 名稱為 etoken 的值，若不存在則為 None。
  • email: Optional[str] = Cookie(None) 亦同理。

問題2：為什麼前端會使用 Cookie 傳遞參數？

Cookie 主要作為維持「用戶狀態」與「身份驗證」的機制：

  • 身份驗證與狀態維護： 儲存 Session ID、Token、Email 等能識別用戶身份的資訊，讓後端能辨識每次請求的用戶。
  • 自動傳遞： 瀏覽器會自動在同網域請求時帶上 Cookie，減少前端每次手動傳遞的負擔。
  • 其他用途： 包含用戶偏好設定、追蹤分析等。

問題3：前端每次呼叫 API 都需要手動設定 Cookie 嗎？

不需要。只要 Cookie 已在瀏覽器儲存（無論是伺服器回應 Set-Cookie 還是前端用 JS 設定），瀏覽器會自動帶上 Cookie 給同網域的請求。

實務測試與注意事項

  • FastAPI 執行測試

  • curl 測試 Cookie 傳遞

  • 注意 Cookie 傳遞條件

    • 必須同網域、同路徑，Cookie 才會自動帶上。
    • 跨網域需考慮 CORS 與 SameSite 屬性。
    • 建議設定 HttpOnly、Secure 屬性提升安全性。
